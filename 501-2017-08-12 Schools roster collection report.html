2017-08-12 Schools roster collection report

Schools roster collection report¶

Provides details on the status of school roster collection

We try to collect the school rosters for 1600ish schools. Schools release their season rosters during the period of July through Feb/May. So, starting in August we run a weekly collection of school rosters (roster collection iteration). With each roster collection iteration we will collect the schools that have published (updated their season roster web page).

Schools by URL_BUILD_CLASS¶

One of the cool things about school roster collection is how the URL of school's season roster is built.

  • For some schools the URL is a simple string with the season specified, so the URL is easy to formulate.
  • For some schools the URL is coded and the codes can be extracted from a season choice list
  • For some schools there is only one URL, and this page needs to detect the season (usually string detect on page)
  • For some schools there is no roster web page, or the roster is not in a web/html table format (can't scrape)

Schools pivot_df¶

The schools pivot_df shows the counts of schools grouped by the way the URL is built so we can detect potential URL problems which can happen when:

  • school changes their web page software or default URL (we have seen a trend of low tech schools upgrading)
  • school changes there URL codes (re-org their sports teams and the internal codes get re-assigned)
  • school has incorrectly configured season choice lists (rare, but NAIA Benedictine has 3 2018 choices, all of which are wrong).

So, looking into the details, we can correct the anomolies, while we watch the majority collect/process, and know our progress.
